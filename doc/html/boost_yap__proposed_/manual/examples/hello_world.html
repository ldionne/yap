Welp. Remember how I mentioned earlier that Boost.YAP does things in a completely lazy way? There's no easy way to get the same line of code that you saw in main() in the Proto version of this example, because Boost.YAP doesn't know that it should evaluate your expression eagerly in this case. It can be done, but it's a bit of code. This is one case in which Proto's eager evaluation can prove easier to use than Boost.YAP's lazy evaluation. 

#include <boost/yap/expression.hpp>

#include <iostream>


int main ()
{
    evaluate(boost::yap::make_terminal(std::cout) << "Hello" << ',' << " world!\n");

    return 0;
}
